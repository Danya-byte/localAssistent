# Security Policy

## Supported versions

Only the latest commit on the default branch and the latest tagged release are supported for security fixes.

## Reporting a vulnerability

Do not open public issues for vulnerabilities or sensitive abuse reports.

Instead:

1. Send a private report to `https://t.me/rollpit`.
2. Include reproduction steps, affected version or commit, and expected impact.
3. If possible, include logs, screenshots, or a minimal proof of concept.

We will acknowledge the report, validate it, and coordinate a fix before public disclosure when appropriate.

## Scope notes

- Model hallucinations or prompt quality problems are usually not security issues by themselves.
- Any bypass of explicit user approval for web, file, or command actions is security-relevant.
- Any unintended file access, command execution, credential exposure, or unsafe installer behavior is security-relevant.
